Projet de stage — 1ʳᵉ année BTS SIO SLAM

Coffre-fort de mots de passe (Python)

Application bureau permettant de stocker et gérer des mots de passe de manière sécurisée : création de compte, mot de passe maître, chiffrement des données et interface graphique pour ajouter, modifier, rechercher et copier les identifiants.

  • Python
  • Interface graphique (Tkinter)
  • Chiffrement symétrique
  • Fichier chiffré vault.enc
  • Architecture modulaire

Contexte du projet

Projet réalisé pendant mon stage de première année de BTS SIO option SLAM. L’objectif était de fournir un petit outil interne permettant de stocker les accès (sites, comptes techniques, outils) de façon plus sécurisée que dans un fichier texte ou un tableur partagé.

L’application fonctionne en local sur le poste de l’utilisateur et repose sur un mot de passe maître pour déverrouiller le coffre-fort.

Objectifs

  • Remplacer les fichiers Excel / blocs-notes par un outil dédié.
  • Protéger les mots de passe par chiffrement fort côté poste client.
  • Proposer une interface simple pour les non-techniques.
  • Faciliter la recherche et la mise à jour des identifiants.

Écran de connexion : création de compte ou connexion avec un mot de passe maître.

Fonctionnalités principales

  • Authentification : création de compte utilisateur et vérification du mot de passe maître (hashé et salé).
  • Gestion des entrées : ajout, modification et suppression de lignes (site, login, mot de passe, catégorie).
  • Recherche rapide : champ de recherche pour filtrer les entrées par site ou login.
  • Générateur de mot de passe : bouton pour créer un mot de passe aléatoire complexe et le remplir automatiquement.
  • Copie sécurisée : bouton pour copier le mot de passe sélectionné dans le presse-papiers.

Toutes les données visibles dans le tableau sont chargées depuis un fichier chiffré et ré-écrites de façon sécurisée après chaque modification.

Interface principale : tableau des mots de passe avec recherche, génération et actions rapides.

Partie technique

Architecture
  • Fichier principal main.py : interface Tkinter et logique métier.
  • Module crypto_utils.py : toutes les fonctions de chiffrement / déchiffrement.
  • Fichiers sensibles :
    • salt.bin : sel utilisé pour dériver la clé à partir du mot de passe maître.
    • vault.enc : contenu du coffre chiffré (sites, logins, mots de passe).
    • vault.env : variables d’environnement liées au coffre (config locale).
Sécurité
  • Utilisation d’une clé dérivée du mot de passe maître avec un sel stocké en binaire.
  • Chiffrement symétrique via une librairie de cryptographie Python.
  • Aucune donnée en clair sur le disque : seul le contenu déchiffré est en mémoire.
  • Mot de passe maître stocké sous forme de hash (pas en clair).

Compétences & apports

  • Mise en place d’une interface graphique complète avec Tkinter.
  • Découverte et utilisation de bonnes pratiques de chiffrement côté client.
  • Organisation d’un projet Python en modules réutilisables.
  • Gestion d’un mini-projet de A à Z : besoin, maquettes, développement, tests.
  • Sensibilisation à la protection de données sensibles en entreprise.

Ce projet m’a permis de faire le lien entre la programmation Python vue en cours et des problématiques très concrètes de sécurité des mots de passe en entreprise.
